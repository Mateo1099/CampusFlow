## 🎯 Directrices Generales

### Principios de Código
- ✅ **Inline CSS solamente** — No usar CSS classes para positioning
- ✅ **Componentes pequeños** — Máx 300 líneas por componente
- ✅ **Estado centralizado** — Usar Context para datos globales
- ✅ **Framer Motion** — Todas las animaciones con Motion, nunca GSAP puro
- ✅ **TypeScript opcional** — JSX está bien, pero DocumentElement tipos ayudan
- ✅ **Sin breaking changes** — Siempre verificar que nada se rompe

### Estructura de Carpetas
```
src/
├── components/        # UI reutilizables
├── pages/             # Rutas principales (Agenda, Dashboard, etc.)
├── context/           # Estado global (Auth, Settings, Tasks)
├── hooks/             # Custom hooks (useTasks, useCourses, etc.)
├── lib/               # Servicios (Supabase, audio, etc.)
└── assets/            # Imágenes y recursos estáticos
```

## 📋 Checklist de Verificación Pre-Commit

Antes de hacer commit de cambios en componentes:

### Componentes UI
- [ ] Modal/form centrado en todos los viewports
- [ ] Colores consistentes (usar CYAN_THEME o variables definidas)
- [ ] Animaciones suaves (spring motion, sin saltos)
- [ ] Responsive en mobile (maxWidth: 95vw mínimo)
- [ ] Scroll funciona si hay overflow
- [ ] Sin console.errors o warnings

### Estado
- [ ] State lifting correcto (no estado duplicado)
- [ ] Props bien tipadas (comentario // PropTypes si es necesario)
- [ ] Callbacks ejecutados en el momento correcto
- [ ] useEffect tiene dependency array correcto

### Integración
- [ ] Context actualizado si hay nuevos estados globales
- [ ] Servicios (Supabase, audio) funcionan correctamente
- [ ] No hay breaking changes en componentes existentes
- [ ] Funciona en navegadores modernos

### Performance
- [ ] No hay re-renders innecesarios (React DevTools Profiler)
- [ ] Animaciones no causan lag (60fps idealmente)
- [ ] Archivos no tienen código muerto

## 🎨 Estándares de Diseño

### Tema Glassmorphism Cyan
**Colores principales:**
```js
const CYAN_THEME = {
  primary: '#00f3ff',           // Cyan brillante
  darkBg: 'rgba(15, 15, 20, 0.95)',     // Fondo muy oscuro
  border: 'rgba(0, 243, 255, 0.35)',    // Borde semi-transparente
  hover: 'rgba(0, 243, 255, 0.15)',     // Hover suave
  selected: 'rgba(0, 243, 255, 0.85)',  // Seleccionado más visible
  shadow: '0 0 30px rgba(0, 243, 255, 0.3)',  // Sombra glow
};
```

**Aplicar a:**
- Modales y formularios
- Inputs y selects
- Botones interactive
- Borders y accents

### Tipografía
- **Títulos:** weight 950, size 1.8rem, text-shadow con cyan
- **Labels:** weight 850, size 0.75rem, UPPERCASE, letter-spacing 0.12em
- **Body:** weight 500, size 1rem, color #ffffff

### Espaciado Estándar
- Modal padding: 40px
- Form gap: 28px entre campos
- Button padding: 12px 32px
- Border radius: 24px (modales), 14px (inputs)

## 🚀 Workflow de Desarrollo

### Crear Nuevo Componente
1. Crear archivo en `src/components/[categoria]/NombreComponente.jsx`
2. Usar inline CSS o CSS modules para estilos
3. Exportar como default export
4. Documentar props (comentario al inicio)

### Agregar Nueva Página
1. Crear archivo en `src/pages/NombrePagina.jsx`
2. Agregar ruta en router (verificar dónde están las rutas)
3. Agregar link en Sidebar si es navegable
4. Verificar que funciona sin breaking changes

### Modificar Estado Global
1. Editar Context en `src/context/`
2. Actualizar useContext en componentes que lo consumen
3. Verificar que no hay ciclos infinitos en useEffect
4. Probar en múltiples componentes

### Integración con Supabase
1. Crear función en `src/lib/[servicio]Service.js`
2. Llamar desde Context o custom hook
3. Manejar errores y loading states
4. Agregar try/catch y logs

## ⚠️ Errores Comunes

| Error | Solución |
|-------|----------|
| Modal descentrado | Consultar `modal-refinement` SKILL |
| Re-renders infinitos | Verificar dependency array en useEffect |
| Estilos no aplican | Inline CSS es case-sensitive (backgroundColor, no background-color) |
| Supabase "no rows returned" | Verificar tabla existe y tiene permisos |
| Audio no suena | Verificar volumen en SettingsContext |
| AnimatePresence no funciona | Debe envolver componente con key prop |

## 📖 Convenciones de Código

### Nombres
- Components: PascalCase (CourseModal.jsx)
- Functions: camelCase (handleDatePickerClick)
- Constants: UPPER_SNAKE_CASE (CYAN_THEME)
